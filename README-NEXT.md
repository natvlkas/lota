# README-NEXT - developer & testing documentation

Active development happens on the **`lota-next`** integration branch.

`main` carries only stable, released code.

This pointer is identical on both branches.

- **Contributing?** Open your pull requests against **`lota-next`**, not `main`.
- **Developer & testing policy** - test layers, how the fuzzers run, and the
  rules for cutting `-rc` release candidates: see
  [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md).
- **Branch model and release flow:** see [docs/BRANCHING.md](docs/BRANCHING.md).
